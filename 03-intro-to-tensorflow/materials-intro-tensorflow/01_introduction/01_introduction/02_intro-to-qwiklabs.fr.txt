Bonjour, bienvenue dans ce cours. Avant d'entrer dans les détails, je voudrais présenter
la plate-forme d'ateliers pratiques appelée Qwiklabs et expliquer
comment vous allez mettre en pratique vos acquis
dans les comptes Cloud Platform réels. Pas besoin de vos identifiants. Nous allons vous en donner. Intéressons-nous
à la plate-forme d'ateliers. Un atelier ou élément est "marqué" pour être ouvert dans un outil externe. Pour cela, cliquez sur "Open Tool". Vous voyez alors un bouton "Start Lab" et une durée qui indique le délai imparti
pour compléter l'atelier. Cliquez dessus.
Les identifiants de votre compte étudiant s'affichent automatiquement
dans une nouvelle boîte. Un bouton jaune "Open Google Console"
s'affiche également. Lorsque vous cliquez dessus, vous êtes invité à vous connecter
à Cloud Platform, en utilisant bien votre compte étudiant et non votre compte personnel,
si vous en avez un. Enfin et surtout, nous allons effectuer des démos et vous présenter l'interface. Allons-y. Dans un module de formation Coursera, un atelier se présente
sous cette forme. Ceci est un exemple, l'atelier 0, mais tous adoptent la même présentation. Après avoir cliqué sur l'atelier, vous devez cocher la case qui s'affiche
pour autoriser l'exécution de l'atelier et accepter les conditions d'utilisation
de Coursera, puis cliquer sur "Open Tool". L'environnement Qwiklabs s'affiche. Ici, il y a trois éléments essentiels. À droite de l'écran, vous pouvez voir
les instructions de l'atelier. C'est un atelier "médaille",
comme la plupart des Qwiklabs. Mais quel que soit le cours,
c'est là que se trouvent les instructions liées à l'atelier. Gardez toujours cette fenêtre ouverte. Lorsque vous cliquez sur "Start Lab", une instance Cloud Platform complète
est provisionnée. Si nécessaire,
des instances complémentaires et/ou des VM sont également provisionnées. Si le créateur de l'atelier souhaite
fournir des ressources, elles sont créées
dans l'instance de l'atelier. C'est une instance "bac à sable". Après avoir cliqué sur "Start Lab", j'ai 15 minutes
pour terminer l'atelier. Nous allons allonger ces délais pour vous, mais vous n'aurez pas besoin de 15 minutes pour compléter cet atelier. Cinq minutes suffisent. Comme je l'ai mentionné, vous avez un nom d'utilisateur,
un mot de passe et un ID de projet. Je vais maintenant vous demander
de cliquer sur "Open Google Console". Vous êtes invité à vous connecter
avec votre adresse e-mail. Si vous avez déjà utilisé
des ressources Google, vous risquez de saisir spontanément
votre e-mail personnel. Revenez à l'écran des identifiants
associés à l'atelier, copiez-collez le nom d'utilisateur
généré automatiquement. C'est un identifiant unique et spécifique à chaque atelier. Nous vous fournissons
un compte étudiant pour chacun. Copiez votre mot de passe. Ne recopiez pas
celui que vous voyez sur mon écran. Cliquez sur "Next" dans votre navigateur,
collez le mot de passe, acceptez les conditions d'utilisation, n'inscrivez rien dans le champ
"Numéro de téléphone secondaire", et enfin, acceptez les conditions
d'utilisation de Google Cloud Platform. Une fois que vous voyez un écran
semblable à celui-ci, tout est prêt. Ceci est le sélecteur de projet.
Vérifiez que la valeur indiquée correspond à l'identifiant alphanumérique
généré pour le Qwiklab. Si la mention "Select a project"
ou "Qwiklab resources" s'affiche ici, cliquez dessus. Ensuite, sélectionnez
l'ID alphanumérique de votre Qwiklab, puis cliquez sur "Open". Vous n'avez plus
qu'à suivre les instructions pour compléter l'atelier. Elles vous présenteront
les produits et services que vous allez utiliser. Vous pouvez accéder au menu
des produits et services en haut à gauche de l'écran. Si vous devez utiliser Cloud Shell,
vous verrez en haut de l'écran le bouton "Activate Google Cloud Shell", qui ouvre une instance de terminal
pour Google Cloud Platform. Autre chose.
Si vous revenez sur cet écran, vous pouvez voir
que le chrono court toujours. Une fois l'atelier achevé,
si vous souhaitez vous arrêter ou si vous voulez recommencer,
notez que vous avez trois tentatives pour chaque atelier. C'est utile. Si vous voulez "survoler" l'atelier, avant de l'explorer plus en profondeur, vous disposez
de deux tentatives supplémentaires. Pour vous assurer
que vos statistiques d'achèvement sont enregistrées, cliquez sur "End Lab". Une fenêtre contextuelle
vous demande une confirmation. Lorsque vous cliquez sur "End Lab",
le projet, les données et les éléments de ce projet
"bac à sable" sont supprimés. Pour conserver certaines requêtes
ou des notes intéressantes de votre compte Qwiklab GCP, copiez-les, puis collez-les
à un autre endroit. Vous pouvez les déplacer
vers votre propre compte GCP, ou sur votre ordinateur. Une fois que vous avez cliqué
sur "End Lab", vous pouvez ajouter
un commentaire sur l'atelier ou passer à un autre cours sur Coursera. Récapitulons. Pour ouvrir les Qwiklabs,
cliquez sur "Open Tool". Pour obtenir les identifiants
à gauche de l'écran, cliquez sur "Start Lab". Cette action
déclenche le chronomètre. Il est alors impossible
de le mettre en pause. Notez que vous disposez
de plusieurs tentatives. Pour quitter un atelier achevé,
cliquez sur "End Lab". C'est tout. Bonne chance
pour la suite, et bon cours !
