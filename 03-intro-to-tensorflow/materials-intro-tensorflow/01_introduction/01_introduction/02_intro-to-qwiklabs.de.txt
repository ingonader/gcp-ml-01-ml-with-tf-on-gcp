Herzlich willkommen beim Kurs! Bevor wir tiefer in die Kurse einsteigen,
zeige ich Ihnen eine Vorschau von Qwiklabs, einer interaktiven, 
praktischen Lab-Plattform, und wie Sie vieles,
was Sie in diesem Kurs lernen, mit Google Cloud Platform-Konten
üben. Dafür sind keine Anmeldedaten nötig. Diese werden zur Verfügung gestellt. Sehen Sie sich den
Aufbau der Lab-Plattform an. In Coursera sehen Sie 
ein markiertes Lab oder Element, über das sich ein externes Tool öffnet. Klicken Sie auf dieses Tool. Sie sehen die grüne Schaltfläche
"Start Lab" und einen Timer mit der Zeit,
die Sie für das Lab haben. Wenn Sie auf "Start Lab" klicken,
sehen Sie in einem Pop-up-Fenster die Anmeldedaten für Ihr Teilnehmerkonto. Außerdem gibt es die gelbe Schaltfläche "Open Google Console". Wenn Sie diese öffnen, werden Sie aufgefordert, sich mit
einem Teilnehmerkonto in der Google Cloud Platform anzumelden und nicht mit Ihrem privaten Konto,
wenn Sie eines haben. Zum Schluss gibt es ein paar Demos und wir zeigen Ihnen, wie die UI aussieht. Werfen wir einen Blick darauf. In einem Coursera-Beispielkurs stoßen Sie vielleicht auf ein Lab,
das ähnlich wie das hier aussieht. Das ist nur ein Beispiel (Lab 0), aber es könnte jedes beliebige Lab sein. Sobald Sie darauf geklickt haben,
sehen Sie ein Kästchen, über das Sie die Lab-Ausführung autorisieren und die
Coursera-Nutzungsbedingungen akzeptieren. Mit einem Klick auf "Open Tool"
öffnen Sie das Tool. Wenn Sie auf "Open Tool" klicken, 
gelangen Sie in die Qwiklabs-Umgebung. Worauf Sie hier achten sollten: Auf der rechten Seite finden Sie
Informationen zum Lab. Dieses Lab ist ein Medaillen-Lab;
es geht um Qwiklabs im Allgemeinen. Egal, um welchen Kurs es geht,
die Anweisungen zum Ausführen des Labs finden Sie immer hier. Diese Meldung sollte
also immer geöffnet sein. Wenn Sie auf "Start Lab" klicken,
wird eine bereitgestellte Instanz der Google Cloud Platform
mit zusätzlichen Instanzen bereitgestellt. Wenn Sie z. B. mit VMs arbeiten oder der
Lab-Anbieter möchte, dass Sie etwas haben, wird dies in dieser
Lab-Instanz für Sie erstellt. Das ist großartig.
Es ist eine Sandbox-Instanz. Wenn ich auf "Start Lab" klicke,
habe ich 15 Minuten dafür Zeit. Die Zeit, die Sie haben, richtet sich 
nach dem zusätzlichen Padding für die Labs. Das heißt nicht,
dass das 15 Minuten dauert, sondern vielleicht 5 Minuten. Wie bereits erwähnt,
haben Sie einen Nutzernamen, ein Passwort und eine Projekt-ID. Klicken Sie auf "Open Google Console". Nach dem Öffnen werden Sie aufgefordert, sich mit Ihrer E-Mail-Adresse anzumelden. Haben Sie bereits Google-Ressourcen
genutzt, geben Sie nicht Ihre private E-Mail-Adresse ein. Gehen Sie
zum Anmeldebildschirm zurück, kopieren sie den automatisch erstellten
Nutzernamen und fügen Sie ihn hier ein. Dieser Name ist für Sie und für jedes Lab,
das Sie ausführen, einmalig. Wir erstellen also für jedes Lab
ein neues Teilnehmerkonto. Kopieren Sie das Passwort
und fügen Sie es ein. Das Passwort ist anders als meines,
Sie können es ohne Bedenken kopieren. Klicken Sie in Ihrem Browser auf "Weiter",
fügen Sie das Passwort ein, bestätigen Sie die Nutzungsbedingungen;
die 2. Telefonnummer brauchen Sie nicht. Stimmen Sie den Nutzungsbedingungen
der Google Cloud Platform zu. Wenn Sie einen Bildschirm sehen, 
der so aussieht, können Sie starten. Achten Sie darauf, dass in der
Projektauswahl diese automatisch generierte, alphanumerische
Qwiklabs-Projekt-ID ausgewählt ist. Wenn Sie etwas wie Projekt auswählen
oder Qwiklabs-Ressourcen sehen, klicken Sie auf die Projektauswahl, wählen Sie Ihre
alphanumerische Qwiklabs-ID aus und öffnen Sie diese. Wenn Sie hier sind, können Sie loslegen. Folgen Sie der Anleitung
bis zum Ende des Labs. Sie werden durch die 
verwendeten Produkte und Dienste geführt. Das Menü für Produkte und Dienste
öffnen Sie in der Ecke links oben. Handelt es sich um ein
Cloud Shell-Lab, öffnen Sie das Menü hier. Hier oben ist die Schaltfläche
zum Aktiveren von Cloud Shell zum Öffnen der Terminalinstanz
für die Google Cloud Platform. Hier sehen Sie, dass unser Timer
noch immer automatisch läuft. Sobald Sie das Lab absolviert haben,
können Sie es abschließen. Wenn Sie das Lab
noch mal absolvieren möchten, Sie haben drei Versuche pro Lab,
können Sie das tun. So können Sie z. B. das Lab
beim ersten Mal sehr schnell absolvieren und beim zweiten Mal mehr
experimentieren und entdecken. Sie können das mehrmals machen. Klicken Sie auf "End Lab",
um Ihren Fortschritt zu speichern. Es wird eine Meldung angezeigt, in der Sie
gefragt werden, ob Sie fertig sind. Wenn Sie auf "End Lab" klicken,
werden Ihr Projekt, Ihre Datasets und alles, womit Sie im Sandbox-Projekt
gearbeitet haben, gelöscht. Speichern Sie alle Abfragen,
die ausgeführt werden, oder interessante Anmerkungen, die in Ihrem
Google Cloud Platform Qwiklabs-Konto gespeichert sind, an einem anderen Ort; entweder in Ihrem privaten
Google Cloud Platform-Konto oder lokal auf Ihrem Rechner. Wenn Sie auf "End Lab" klicken,
können Sie Feedback zum Lab geben und weitere Coursera-Kurse absolvieren. Noch mal zusammengefasst:
Was Sie wissen müssen, ist, dass Sie auf das Tool klicken,
mit dem Qwiklabs geöffnet wird. Um die Anmeldedaten zu erhalten,
müssen Sie hier auf der linken Seite auf "Start Lab" klicken.
Damit wird auch der Timer gestartet. Sie können das Lab nicht pausieren, aber Sie können es mehrmals ausführen. Zum Schluss klicken Sie auf "End Lab", um das Lab abzuschließen. Das wars. Viel Glück mit den
anderen Labs und viel Spaß beim Kurs!
