Bem-vindo de volta! Sou Evan Jones,
desenvolvedor de currículo técnico do Google Cloud e entusiasta do Big Data. Ainda sobre aprendizado de máquina (ML)
com generalização e amostragem. Até agora, discutimos o treinamento
do modelo de ML e testamos o treinamento do modelo no 
Tensor Flow Playground. Agora é hora de responder
a uma pergunta bem estranha. Quando o modelo de ML
mais preciso não é o ideal? E, como sugerimos no último
módulo sobre otimização, ter um modelo com métrica de perda zero
para seu conjunto de dados de treinamento não significa bom desempenho
em novos dados no mundo real. O melhor modelo de ML não é
necessariamente aquele que tem o melhor desempenho em seu
conjunto de dados de treinamento, mas o que tem melhor desempenho
em dados ainda não vistos. Sua principal preocupação deve ser o 
desempenho do modelo em produção. E isso envolve dados que o seu
modelo ainda não conhece. Como seria o desempenho do seu modelo
com dados desconhecidos? Primeiro, obtenha alguns dados
que não são exibidos no modelo durante o treinamento. E, depois de treinar o
modelo com êxito, você poderá avaliá-lo neste
conjunto de dados desconhecido. Você aprenderá a avaliar se o modelo
está se ajustando demais ou não e quando parar de fato
o treinamento do modelo. A segunda parte deste módulo fala
sobre como criar este conjunto de dados desconhecidos antes de tudo. Obviamente, você não tem
dados desconhecidos, mas um conjunto
que pode ser dividido em conjuntos de dados
de treinamento e avaliação. Teste e treine seu modelo 
com um conjunto de dados. E, quando estiver pronto,
meça o desempenho do modelo no mundo real em relação a um conjunto 
de dados de avaliação ou teste. Assim, você aprende a criar
conjuntos de dados repetíveis para teste e avaliação do treinamento e estabelece
comparativos de desempenho.
