Nun ein Blick auf die
Entwicklungsumgebung, die Sie für die Spezialisierung verwenden. In diesem Modul geht es um
Python-Notebooks in der Cloud. Sie erfahren, wie Sie ML-Modelle in
Python-Notebooks entwickeln, wo sich der Notebook-Server
in der Cloud befindet und wie Sie Ad-hoc-Abfragen mit
serverlosen Technologien ausführen – eben über diese Python-Notebooks. Sie müssen ML-Modelle nicht
immer von Grund auf neu erstellen. In diesem Modul erfahren
Sie von meiner Kollegin Sarah, wie Sie vortrainierte ML-Modelle aufrufen. Cloud Datalab ist die
integrierte Entwicklungsumgebung, die Sie in diesem Kurs zum
Schreiben von Code verwenden. Cloud Datalab-Notebooks werden
auf virtuellen Maschinen ausgeführt. Daher erläutern wir auch
Compute Engine und Cloud Storage. Warum? Die Ausführung auf einer
VM ist mit zwei Dingen verbunden: Erstens: Sie können bestimmen, auf welchem Gerät das
Notebook ausgeführt wird, indem Sie beispielsweise mehr
Speicher oder GPO hinzufügen, ohne das Notebook neu schreiben zu müssen. Das erneute Hosten auf einem
schnelleren System ist denkbar einfach. Zweitens: VMs sind sitzungsspezifisch. Alle Daten, die Sie beibehalten möchten, müssen außerhalb der virtuellen
Maschine gespeichert werden. Der beste Ort hierfür,
insbesondere für große Binärdateien, ist Cloud Storage. Nachdem wir uns mit
Compute Engine beschäftigt haben, werden wir uns die Grundlagen
von Cloud Storage anschauen. Die Notebooks selbst werden
in einem Cloud-Repository gespeichert. Damit unterliegen sie
der Versionsverwaltung. Abschließend ein praxisorientiertes Lab,
damit Sie sich mit Datalab vertraut machen. Wie zeigen Ihnen, wie Sie Cloud Datalab
zusammen mit BigQuery verwenden können: ein verwalteter Datenanalysedienst in
der Cloud, mit dem Sie Ad-hoc-Abfragen mit einer Geschwindigkeit und in
einem Umfang ausführen können, die traditionelle Datenbanksysteme
nicht bieten können. Dann erfahren Sie, wie Sie vortrainierte ML-Modelle
in Cloud Datalab aufrufen können.
