Lo que acabamos de hacer
fue crear una instancia de una VM de Compute Engine solo con el propósito
de mostrarles algunos scripts. Pero es una gran pérdida de tiempo. En realidad, no necesitan hacerlo. Les mostraré algo genial. Si van a GCP Console,
verán este pequeño ícono que dice "Activate Google Cloud Shell". Hagan clic en él. Y tendrán un servicio
que se llama Cloud Shell que es una micro VM. Es una VM muy pequeña. Pero, por lo general,
es suficiente para nuestros propósitos si todo lo que queremos hacer
es probar algunos scripts copiar algunos archivos
de una ubicación a otra. Estamos en Cloud Shell
y puedo hacer casi lo mismo que hice en la VM de Compute Engine. Excepto que Cloud Shell
está diseñada para los desarrolladores. Mucho de lo que tuvimos
que instalar en la VM de Compute Engine no necesitamos hacerlo aquí. Por ejemplo, Git ya está instalado. Si quieren usar Git,
simplemente usarían git clone... y clonarían un repositorio.
Ya está. Ahora tenemos todo el código
que necesitábamos y podemos hacer gran parte del lab
que hicimos en Compute Engine en Cloud Shell. Lo que deben saber de Cloud Shell
es que es una VM muy efímera. Si no la usan durante menos de una hora,
se la recicla. Pero pueden iniciar
otra instancia de Cloud Shell Pueden usar otra pestaña. Es la misma VM,
pero pueden tener dos pestañas en ella. Y pueden ejecutar otros comandos
en esta pestaña. Usaremos Cloud Shell
para tareas como iniciar Datalab. Es lo que harán en el siguiente lab.
