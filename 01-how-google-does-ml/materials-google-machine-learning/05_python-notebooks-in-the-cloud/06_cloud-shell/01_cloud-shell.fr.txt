Nous venons de créer
une VM Compute Engine. simplement pour lancer
quelques scripts. C'est du temps de perdu. Ce n'est pas nécessaire. Je vais vous montrer
une chose intéressante. Si vous ouvrez la console GCP, vous verrez cette icône qui permet d'activer
Google Cloud Shell. Cliquez dessus. Vous activez ainsi
l'environnement Cloud Shell. Il s'agit d'une micro-VM, une toute petite VM. Mais c'est suffisant pour nos besoins, si nous nous contentons
de faire quelques essais, d'exécuter quelques scripts, de copier quelques fichiers
d'un emplacement vers un autre. Nous sommes donc dans Cloud Shell, et je peux donc faire presque tout ce que je pourrais faire
avec une VM Compute Engine. Mais Cloud Shell est conçu
pour les développeurs. De nombreux éléments que nous avions dû
installer sur la VM Compute Engine le sont déjà ici. Par exemple, Git est déjà là. Pour accéder à Git, utilisez la commande "git clone". Vous pouvez alors cloner un dépôt. Nous disposons du code
dont nous avons besoin. Nous pouvons faire presque tout l'atelier
comme dans Compute Engine. On peut se servir de Cloud Shell. Notez cependant que Cloud Shell
est une VM éphémère. Si vous ne l'utilisez pas pendant une heure, il est recyclé. Mais vous pouvez redémarrer Cloud Shell. Vous pouvez ouvrir
un autre onglet dans Cloud Shell. Il s'agit de la même VM, mais elle compte maintenant deux onglets. Vous pouvez exécuter
d'autres commandes dans cet onglet. Nous allons utiliser Cloud Shell
par exemple pour lancer Datalab. C'est l'objectif du prochain atelier.
