Gerade haben wir
eine Compute Engine-VM erstellt, nur um einige Skripts
mit Branding zu versehen. Das war reine Verschwendung. So etwas ist unnötig. Ich zeige Ihnen jetzt
etwas wirklich Tolles. Wenn Sie die GCP Console öffnen, sehen Sie dieses kleine Symbol: "Google Cloud Shell aktivieren". Klicken Sie darauf. Daraufhin wird Cloud Shell geöffnet. Cloud Shell ist eine Micro-VM, also eine winzig kleine VM. Doch für unseren Bedarf reicht sie, wenn es nur darum geht,
etwas auszuprobieren, einige Skripts auszuführen oder ein paar Dateien
an einen anderen Ort zu kopieren. Das ist also Cloud Shell. Damit kann ich praktisch dasselbe tun wie mit der Compute Engine-VM. Cloud Shell ist aber
für Entwickler gedacht. Bei der Compute Engine-VM
mussten wir erst vieles installieren. Das brauchen wir hier nicht. Zum Beispiel ist Git bereits vorhanden. Wenn Sie also Git nutzen möchten, können Sie einfach "git clone" eingeben. Klonen Sie dann unser Repository. Damit haben wir
den gesamten Code, den wir brauchen, und können die gleichen Aufgaben
ausführen wie vorhin mit Compute Engine. Alles ist in Cloud Shell möglich. Allerdings ist Cloud Shell
sitzungsspezifisch. Bleibt es inaktiv, wird es meist
nach maximal einer Stunde recycelt. Doch dann können Sie
eine neue Cloud Shell anlegen. Cloud Shell bietet mehrere Tabs. Es ist immer noch dieselbe VM, aber jetzt mit zwei Tabs. Auf diesem Tab
können Sie andere Befehle ausführen. Wir nutzen Cloud Shell hier,
um beispielsweise Datalab zu starten. Damit befassen wir uns im nächsten Lab.
