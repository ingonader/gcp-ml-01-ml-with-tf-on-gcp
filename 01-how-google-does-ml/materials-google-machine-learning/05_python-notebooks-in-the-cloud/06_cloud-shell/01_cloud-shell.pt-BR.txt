O acabamos de fazer aqui foi criar uma
VM do Compute Engine só para registrar alguns scripts. Isso é um grande desperdício. Não precisamos fazer isso. Deixe-me mostrar algo muito legal. Se você for ao console do GCP, verá esse ícone pequeno que diz: "Ativar o Google Cloud Shell". Clique nele. Você terá algo chamado Cloud Shell. E o Cloud Shell é uma micro-VM. É uma VM muito pequena. Mas normalmente é suficiente para os
nossos propósitos se tudo o que quisermos fazer for
experimentar algumas coisas, executar alguns scripts, copiar alguns arquivos de um local
para outro. Aqui estamos no Cloud Shell, e podemos seguir em frente e fazer as mesmas coisas que fizemos na VM
do Compute Engine. Exceto pelo fato de o Cloud Shell ser para
desenvolvedores. Por isso, muitas das coisas que tínhamos
que instalar na VM do Compute Engine, não instalaremos aqui. O Git, por exemplo. Já está aqui. Se você quiser ter o Git, é só digitar "Git clone". E clonar um repositório. Aqui vamos nós. Agora temos todo o código que
precisávamos ter, e podemos fazer muito do que fizemos em
nosso laboratório no Compute Engine. É só fazer isso no Cloud Shell. O Cloud Shell é uma VM muito temporária. Se não usá-la, normalmente em até uma hora,
ela será reciclada. Mas você sempre pode iniciar
outro Cloud Shell. Você pode abrir outra guia no Cloud Shell. Ainda é a mesma VM, mas agora você tem duas guias nela. E você pode executar outros comandos
nesta guia. Usaremos o Cloud Shell como uma forma de
fazer coisas como iniciar o Datalab. É algo que faremos no próximo laboratório.
