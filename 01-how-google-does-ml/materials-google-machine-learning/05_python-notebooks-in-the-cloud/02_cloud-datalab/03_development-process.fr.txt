Avez-vous déjà utilisé Google Docs ? En quoi est-ce différent des documents
modifiés dans un éditeur de bureau ? Avez-vous déjà rempli
une déclaration d'impôts en ligne ? En quoi est-ce différent
de la faire dans un logiciel de bureau ? De tous les avantages,
la collaboration est le plus important. Pas besoin d'envoyer et de 
renvoyer les documents par e-mail. Vous pouvez faire votre déclaration
en collaboration avec votre comptable. Lorsque j'ai commencé
à faire de la recherche scientifique, collaborer n'était pas une mince affaire. Une fois le code et le graphique créés,
il me fallait faire une capture d'écran, créer une image à copier dans un document,
puis un PDF à envoyer à ma collaboratrice. Quelques heures plus tard,
elle me répondait : "C'est bien, mais pourrais-tu ajouter une année ? Ça fait un peu léger." Et je recommençais
le processus de zéro. Pourquoi ? Car le PDF envoyé
n'était pas modifiable. Ces échanges prenaient du temps. C'est alors que le bloc-notes IPython
a fait son entrée. Je créais alors le code et un graphique,
puis je rédigeais quelques commentaires, avant d'envoyer le lien du bloc-notes
à ma collègue. Lorsqu'elle voulait ajouter une année,
il lui suffisait de modifier la cellule, d'examiner
le nouveau graphique et de se féliciter. C'était parfait. Nous disposions d'un bloc-notes
plus performant. Mais ce modèle traditionnel
posait problème. Qui exécute le serveur
qui héberge ces pages ? Avec quelle machine ? Le mien ? S'il se met en veille,
ma collègue ne peut pas travailler. Mais lorsque votre bloc-notes Datalab
est hébergé dans le cloud, vous pouvez collaborer sans problème. Google Docs est disponible
même lorsque votre ordinateur est éteint. Il en va de même pour les blocs-notes
Datalab exécutés sur le cloud. Pour partager un bloc-notes
au sein d'un projet, les utilisateurs n'ont qu'à se connecter
à la VM et à travailler à l'aide de l'URL. Ils peuvent aussi utiliser les systèmes
de contrôle des révisions tels que Git. Le cloud vous permet de ne pas être freiné
par les limitations des logiciels. Vous pouvez non seulement exécuter Datalab
sur toute instance de VM de votre choix, mais aussi modifier les spécifications
de la machine après l'avoir provisionnée. Vous pouvez consulter la console Web,
trouver la VM utilisée, l'arrêter, puis la redémarrer
avec une nouvelle configuration. En tant que développeur,
pour utiliser Datalab, il vous suffit de connecter
la VM exécutant le serveur du bloc-notes. Les blocs-notes
peuvent être conservés dans Git. Vous pouvez ainsi supprimer
la VM si vous n'en avez plus besoin. Lorsque vous exécutez le bloc-notes,
le calcul est effectué sur la VM.
