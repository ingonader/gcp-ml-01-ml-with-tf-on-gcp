Se tem algo que eu quero
que você aprenda neste módulo é que a magia do aprendizado de máquina
é a quantidade, e não a complexidade. Muitos pequenos modelos
de aprendizado de máquina, todos simples. Então, pense no exemplo do Maps. Se você está criando muitos modelos
de aprendizado de máquina e planejando muitos outros
que talvez nunca crie, você precisa ter um ambiente
para testar e falhar. A ideia é que
se você está falhando rápido, você vai poder iterar. A capacidade de experimentar
é essencial para o aprendizado de máquina. É importante falhar logo e iterar
para que possa testar muitas ideias novas, e dessa forma, ter mais sucesso. Quando eu falo em dados, a maioria das pessoas pensa
imediatamente em dados estruturados, como dados em um banco de dados. Mas 90% dos dados corporativos
não são estruturados. Pense em e-mails, vídeos,
textos, relatórios, catálogos, sessões de moda, eventos, notícias, qualquer coisa. Todos dados não estruturados. Lidar com dados não estruturados ficou muito mais fácil por causa dos modelos
pré-treinados sobre os quais falamos. Então, pense em um canal de ML como uma
forma de lidar com dados não estruturados. Pegue os dados não estruturados,
passe-os pelas ML APIs, e então você terá entidades, lugares, rótulos, pessoas, coisas com as quais
você pode criar um modelo de ML simples. Então não é preciso
começar com dados não estruturados, processe os dados não estruturados
por meio de uma ML API e pegue os resultados dessa API como a entrada
para seu modelo de ML personalizado.
