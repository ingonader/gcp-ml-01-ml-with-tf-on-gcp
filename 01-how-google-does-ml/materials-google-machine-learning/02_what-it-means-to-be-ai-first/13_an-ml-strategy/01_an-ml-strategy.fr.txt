Ce que je veux que vous reteniez
après avoir suivi ce module, c'est que la magie du ML est une question de quantité
et non de complexité. Il faut créer un grand nombre
de petits modèles de ML simples. C'est comme une carte, par exemple. Si vous créez beaucoup de modèles et en prévoyez autant
que vous ne concevrez peut-être jamais, il vous faut
une méthode de création axée sur l'échec. En effet, plus vous échouez,
plus vous pouvez recommencer. Les tests sont une étape cruciale
dans le domaine du ML. Vous devez donc échouer
et recommencer souvent pour essayer un maximum de nouvelles idées et, ainsi, obtenir
un plus grand nombre de réussites. Quand je parle de données, la plupart des gens pensent
aux données structurées, comme celles d'une base de données. Mais 90% des données d'une entreprise
ne sont pas structurées. Pensez aux e-mails, aux vidéos,
aux documents écrits, aux rapports, aux catalogues, aux événements,
aux shootings de mode, aux actualités, etc.
Ce sont des données non structurées. Leur traitement s'est beaucoup simplifié grâce aux modèles pré-entraînés
dont nous avons déjà parlé. Considérez donc un pipeline de ML comme le moyen
de traiter des données non structurées. Analysez vos données non structurées
via des API de machine learning et vous obtenez des entités, des lieux,
des libellés, des personnes, etc., c'est-à-dire des données avec lesquelles
vous pouvez créer un modèle de ML simple. Inutile, donc, d'utiliser
les données non structurées. Analysez-les
via une API de machine learning et utilisez
les données structurées générées comme données d'entrée
pour votre modèle personnalisé.
