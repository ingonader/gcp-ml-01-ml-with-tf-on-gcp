Ich möchte, dass Sie sich eine
Sache aus diesem Modul merken: Bei maschinellem Lernen
geht Quantität über Komplexität. Es gibt viele kleine Modelle, die alle ganz einfach sind. Stellen Sie sich eine Karte vor: Wenn Sie viele Modelle für
maschinelles Lernen erstellen und viele weitere planen,
die vielleicht nie zustande kommen, dann wollen Sie schnell
wissen, wann Sie gescheitert sind. Nur dann können Sie die Modelle weiterentwickeln. Experimentieren ist extrem
wichtig bei maschinellem Lernen. Scheitern Sie früh, 
können Sie viele neue Ideen ausprobieren und noch erfolgreicher werden. Wenn ich von Daten spreche, denken die meisten
gleich an strukturierte Daten in einer Datenbank. Aber 90 % aller
Unternehmensdaten sind unstrukturiert. So auch E-Mails, Videos, SMS, Berichte, Kataloge, Mode-Shootings, Events, Nachrichten usw. Alles unstrukturierte Daten. Der Umgang mit diesen Daten ist heute dank
vortrainierter Modelle viel einfacher. Mit ML-Pipelines können
unstrukturierte Daten bearbeitet werden. Wenn diese Daten ML-APIs durchlaufen, bleiben Entitäten, Orte,
Labels und Menschen übrig. Alles Dinge, aus denen Sie
ein einfaches ML-Modell erstellen können. Sie müssen nicht mit
unstrukturierten Daten beginnen. Verarbeiten Sie die Daten
mit einer ML-API und verwenden Sie die Ergebnisse zur Eingabe für
Ihr benutzerdefiniertes ML-Modell.
