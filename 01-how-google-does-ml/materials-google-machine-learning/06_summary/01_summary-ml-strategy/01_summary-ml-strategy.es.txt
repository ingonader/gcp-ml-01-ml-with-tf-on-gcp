Hagamos un resumen del curso. Como dijimos,
el objetivo de la especialización es enseñarle a generar
modelos de AA de producción. No importa si es un programador
de Python o un científico de datos. La especialización
consta de varios cursos que brindan una introducción al AA
de manera práctica y realista. Luego, vimos
la perspectiva de Google sobre el AA como reemplazo para las reglas heurísticas
que tienden a acumularse con el tiempo. Por ejemplo, la Búsqueda de Google
usaba reglas heurísticas para decidir si necesitábamos mostrar una página
de los Gigantes de San Francisco o de NY. Ahora eso se determina
con aprendizaje automático. Luego, hablamos
sobre el aprendizaje automático como una manera de personalizar
lo que ofrece su empresa a los clientes. Usamos Google Maps como ejemplo. Vimos que se puede trazar una ruta
entre A y B con un algoritmo determinista, pero para inferir en qué piso
de una estación de metro está el usuario se necesita al menos
un poco de aprendizaje automático. Y si queremos generar
recomendaciones personalizadas a escala, nos enfrentamos a un desafío imposible
si no usamos el aprendizaje automático. Luego, profundizamos
en cómo funciona el AA. Y resulta que no tiene nada de magia. La clave es desarrollar
conocimiento práctico organizativo. Es importante reconocer las 5 fases
que atraviesan los procesos empresariales. No debe intentar pasar por alto
ninguna de ellas para triunfar en el AA. También es importante saber
cómo pasar de una fase a otra. Por último, hablamos de por qué
el AA no es siempre "justo" y de cómo reconocer
inclinaciones que se pueden amplificar debido a los datos usados. Las inclinaciones inconscientes
afectan la manera en la que usamos el AA, ya que pueden reforzarse con los datos
de entrenamiento que recopilamos. Afectan nuestra forma
de recopilar y clasificar los datos y de diseñar y escribir código. Como primera experiencia
de AA, presentamos Qwiklabs para que pudiera probar
Google Cloud Platform. Esto le permite hacer
los labs de esta especialización sin pagar un adicional
por los recursos de procesamiento. Qwiklabs también
tiene otros labs y quests. Le recomiendo que lo use
para practicar y ganar experiencia en el aprendizaje automático
y las tecnologías de la nube. También vimos
cómo buscar el código fuente. Está en GitHub, es código abierto, y recomendamos que use nuestros ejemplos
como punto de partida para sus proyectos. También hablamos de Cloud Datalab. Los cuadernos de Python
son una herramienta favorita de los científicos de datos. Los usará para la mayoría
de las actividades de la especialización. En esta especialización,
los cuadernos ya están resueltos, por lo que recomiendo que se detenga
y piense en cada paso de los cuadernos. Si cree estar listo,
puede quitar algunas celdas clave para ver si puede escribir
el código necesario por su cuenta. Mencionamos
que Cloud Storage y Compute Engine brindan la potencia de CPU
y el almacenamiento que se requieren para los cuadernos distribuidos efímeros. Luego, usamos un cuaderno
para lanzar consultas de BigQuery en miles de máquinas a escala. Por ejemplo, esta es una consulta
que usa un conjunto de datos de millones de reclamaciones de salud. También invocamos modelos
de AA ya entrenados, disponibles como API. A medida que evolucione
el aprendizaje automático, muchas de las tareas reutilizables estarán disponibles en formatos
ya entrenados, ya sea en Vision o Speech. Un punto clave
para aprender de estas API de AA es que nuestros modelos de AA
deberían ser igual de fáciles de usar. En este caso, todas las API
de AA eran API de REST, microservicios con un nivel
de abstracción muy elevado. Aquí finaliza
el primer curso de la especialización. En el próximo, hablaremos
sobre cómo crear conjuntos de datos de AA.
