Então, para resumir este curso falamos sobre o
objetivo da especialização que é ensinar você a criar modelos de aprendizado
de máquina para produção mesmo que seja um programador
Python ou um cientista de dados. Essa especialização consiste
em uma série de cursos que fornecem uma introdução prática
ao ML em cenários reais. Depois, vimos que o Google
considera o aprendizado de máquina como uma forma de substituir as regras
heurísticas criadas ao longo do tempo. Por exemplo, a Pesquisa Google
usava regras heurísticas para decidir se era preciso retornar uma página do San Francisco
Giants ou do New York Giants. Agora isso é feito usando
aprendizado de máquina. Em seguida, falamos que o
aprendizado de máquina é um jeito de personalizar
as ofertas para seus clientes usando o Google Maps, por exemplo. Ao traçar rotas entre
os pontos A e B, por exemplo, é possível usar um algoritmo
determinístico que deduza o andar da estação de metrô que
você está, usando pouco ML. Mas é impossível criar
recomendações personalizadas em grande escala sem usar ML. Depois, falamos sobre os segredos
do aprendizado de máquina. Eles não são nada glamorosos. O segredo é conhecimento organizacional. É importante reconhecer as
cinco etapas dos processos de negócios, e nem pense em pulá-las se quiser
criar um aprendizado de máquina. Também é importante entender
como fazer a transição entre elas. Por último, falamos por que
o aprendizado de máquina não mostra resultados justos
e como reconhecer vieses que podem ser ampliados
dependendo dos dados usados. Os vieses inconscientes
afetam o aprendizado de máquina porque podem ser reforçados
nos dados de treinamento coletados e afetam a coleta
e a classificação de dados e a programação dos códigos. Para mostrar um
pouco mais sobre ML ensinamos você a usar o
Google Cloud Platform pelo Qwiklabs. Esse é um jeito de fazer os
laboratórios desta especialização sem precisar pagar
pelos recursos computacionais. O Qwiklabs também tem
vários outros laboratórios e quests. Recomendo que os pratique e ganhe mais experiência em ML
e tecnologias do Cloud com o Qwiklabs. Também mostramos como
encontrar o código-fonte no GitHub que é um recurso de código aberto. Use nossos exemplos como um
ponto de partida para seus projetos. Depois, vimos o Cloud Datalab. Os blocos de anotações do Python
são a opção de vários cientistas de dados, e você pode usá-los na maioria das
atividades práticas desta especialização. Os blocos de anotações desta
especialização estão prontos para uso. Recomendo que você
analise os blocos com atenção. Se você aceitar o desafio poderá remover algumas células e ver se consegue
escrever os códigos sozinho. Falamos sobre como o Cloud Storage
e o Compute Engine fornecem a CPU e o armazenamento para blocos
de anotações distribuídos temporários. Depois, usamos o bloco para inicializar consultas em várias máquinas
em grande escala no Big Query. Por exemplo, essa é uma
consulta que pode ser feita em conjuntos de dados de pedidos
de indenização em planos de saúde. Também falamos sobre os modelos
pré-treinados disponíveis como APIs. Como resultados do
aprendizado de máquina muitas das tarefas reutilizáveis estarão
disponíveis de maneira pré-treinada seja na Vision API ou na Speech API. Com essas APIs de aprendizado de máquina aprendemos que os modelos
de ML precisam ser fáceis de usar. Nesse caso, todas as APIs
de ML e as REST APIs são microsserviços e
fornecem um alto nível de abstração. Assim, chegamos ao fim
do primeiro curso da especialização. Esperamos você no próximo curso
que falará sobre conjuntos de dados de ML.
