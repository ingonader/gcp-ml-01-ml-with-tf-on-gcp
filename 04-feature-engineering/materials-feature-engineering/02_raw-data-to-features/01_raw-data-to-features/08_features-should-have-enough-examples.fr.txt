Passons au quatrième point. Vous devez avoir assez d'exemples
de chaque valeur d'une caractéristique dans votre ensemble de données. Nous utilisons une bonne pratique
qui consiste à avoir au moins cinq exemples pour chaque valeur
avant d'utiliser la caractéristique. Vous devez avoir cinq exemples
pour la valeur dans vos données avant de démarrer l'entraînement. Qu'est-ce que cela signifie ? Prenons un exemple. Si vous avez une catégorie d'achat
qui correspond aux automobiles, vous devez avoir assez de transactions
frauduleuses ou non d'achats d'automobiles, pour évaluer les transactions frauduleuses
dans votre modèle. Si vous n'avez que trois transactions
dans votre ensemble de données, et qu'elles sont toutes non frauduleuses, le modèle apprendra
que personne ne commet jamais de fraude sur des automobiles. C'est logique, puisque vous n'avez
aucune transaction frauduleuse. Ce sera un gros problème. Il faut donc éviter ce type de problème
dû au manque d'exemples. Je ne dis pas que vous devez
avoir au moins cinq catégories, mais au moins cinq exemples. Pour chaque valeur d'une colonne,
vous devez avoir ces cinq exemples. Revenons à notre exemple d'allée fissurée
dans notre modèle immobilier précédent. Si vous pensez qu'une photo d'allée fissurée
est un bon indicateur du prix d'une maison, assurez-vous d'avoir assez d'exemples
pour entraîner votre modèle.
